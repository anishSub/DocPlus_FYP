OVERVIEW
--------
Implemented a real-time chat system for video consultations using Django 
Channels and WebSockets. This allows doctors and patients to communicate 
instantly during video calls without page reloads.

Key Features:
- Instant bidirectional messaging (Doctor ↔ Patient)
- No page reload required
- Persistent chat history (saved to database)
- Appointment-specific chat rooms (isolated)
- Automatic reconnection on connection loss
- Authentication and access control

===============================================================================
HOW THE SYSTEM WORKS
===============================================================================

MESSAGE FLOW:
------------
1. User opens video call page
2. JavaScript establishes WebSocket connection to ws://localhost:8000/ws/chat/<id>/
3. Backend verifies user is authorized (doctor or patient of that appointment)
4. Backend sends chat history (previous messages)
5. User types message and clicks Send
6. JavaScript sends message via WebSocket
7. Backend receives message, saves to database
8. Backend broadcasts message to all connected users in that room
9. Both doctor and patient receive message instantly
10. Message appears in chat sidebar (no page reload)

SECURITY:
--------
- Only authenticated users can connect
- Only doctor and patient of specific appointment can access chat
- Each appointment has isolated chat room
- Messages saved to database for persistence
- XSS protection via HTML escaping

===============================================================================
PRODUCTION DEPLOYMENT NOTES
===============================================================================

For production, you need to:

1. SWITCH TO REDIS CHANNEL LAYER:
   In settings.py:
   CHANNEL_LAYERS = {
       'default': {
           'BACKEND': 'channels_redis.core.RedisChannelLayer',
           'CONFIG': {
               "hosts": [('127.0.0.1', 6379)],
           },
       },
   }

   Install Redis:
   - macOS: brew install redis
   - Ubuntu: sudo apt-get install redis-server
   - Start: redis-server

2. RUN WITH ASGI SERVER:
   Using Daphne:
     daphne -b 0.0.0.0 -p 8000 myproject.asgi:application
   
   Or using Uvicorn:
     uvicorn myproject.asgi:application --host 0.0.0.0 --port 8000

3. CONFIGURE HTTPS FOR WSS:
   WebSocket Secure (wss://) required for HTTPS sites
   Update Nginx/Apache configuration to proxy WebSocket connections

4. OPTIONAL ENHANCEMENTS:
   - Add rate limiting (prevent spam)
   - Add read receipts (mark messages as read)
   - Add typing indicators
   - Add message deletion
   - Add file sharing in chat


===============================================================================
TROUBLESHOOTING
===============================================================================

ISSUE: WebSocket connection failed
SOLUTION:
  - Ensure server is running
  - Check if Channels is installed: pip list | grep channels
  - Verify ASGI configuration in settings.py
  - Check browser console for error details

ISSUE: Messages not appearing
SOLUTION:
  - Check WebSocket connection status in console
  - Verify user is authenticated
  - Check Django server logs for errors
  - Verify appointment ID is correct

ISSUE: Chat history not loading
SOLUTION:
  - Check database for ChatMessage records
  - Verify get_chat_history() method in consumers.py
  - Check browser console for received data
